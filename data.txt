What drives DP vs AC

DP is not set directly from app data at insertion time; it first depends on mSPAppStatus. That variable is populated in validApplicationCheck() for SSA MSP apps:

If QuestionThreeSw == 'Y' → mSPAppStatus = "DP"

Else if ExtHelpAprSw == 'Y' and FplIncomePercent > 150 → mSPAppStatus = "DP"

Else if ExtHelpAprSw != 'Y' and FplIncomePercent > 135 → mSPAppStatus = "DP"
If none of those happen, mSPAppStatus stays null, and later the AR app row gets "AC" instead.
